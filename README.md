# Seek-IN Analytics

**Team Binaries** 

A secure, conversational AI system that lets non-technical business stakeholders ask questions about their company database in plain English — without ever exposing the real database schema to an external AI service.

Built for the Bharat Academix Hackathon, Round 2 (Prototype Development).

---

## What it does

1. A user types a question like *"Show me customers from Lucknow"*.
2. The real database schema is **disguised** — real table/column names like `customers.name` are aliased to meaningless names like `table_A.col_A2` before anything is sent to an AI.
3. An AI model (Llama 3.3, via Groq) reads the question and the **disguised** schema, and writes a SQL query using only the fake names.
4. The fake names are converted back to real ones, the query is checked for safety (only `SELECT` statements are allowed — no `DROP`, `DELETE`, `UPDATE`, etc.), then run on the real MySQL database.
5. The raw results are turned into a plain-English answer and shown to the user.

At no point does the AI ever see a real column name, table name, or piece of business data structure.

## Tech stack

- **Backend:** Python, Flask
- **AI:** Groq API (Llama 3.3 70B)
- **Database:** MySQL
- **Frontend:** HTML, CSS, JavaScript

## Project structure

```
Seek-In/
├── Backend/
│   ├── app.py                 # Main Flask app -- connects all pieces together
│   ├── obfuscation.py         # Aliases real schema names <-> fake names
│   ├── nl_to_sql.py           # Sends question + fake schema to AI, gets SQL back
│   ├── validator.py           # Blocks any non-SELECT / unsafe SQL
```

## Setup instructions

### 1. Clone the repo
```
git clone https://github.com/sarthak232007/Seek-In.git
cd Seek-In
```

### 2. Create a virtual environment
```
python -m venv venv
venv\Scripts\activate        # Windows
```

### 3. Install dependencies
```
pip install flask flask-cors mysql-connector-python python-dotenv groq
```

### 4. Set up the database
Open MySQL and run:
```
mysql -u root -p
source Database/schema.sql
source Database/seed_data.sql
```

### 5. Set up environment variables
Copy `.env.example` to a new file named `.env` in the `Backend` folder, and fill in your real values:
```
GROQ_API_KEY=your_groq_api_key_here
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=seekin_demo
TEST_MODE=false
```

You can get a free Groq API key at [console.groq.com](https://console.groq.com).

### 6. Run the backend
```
cd Backend
python app.py
```

### 7. Open the frontend
Open `Frontend/index.html` directly in your browser.

## Known limitations (and what production deployment would add)

This is a hackathon prototype. For a real production deployment, we would add:

- **Automatic schema introspection** — currently the alias map (`schema_config.json`) is written manually for our demo database. In production, this would be auto-generated by reading the connected database's actual structure.
- **A proper SQL parser** for the obfuscation layer instead of text-replacement, to safely handle complex queries with repeated column names across tables.
- **Per-business database connections** — each business would connect their own database; Seek-IN does not store or share data across businesses.
- **Billing-tier AI usage** for guaranteed higher rate limits and stronger data-privacy terms in production.

See `docs/architecture.md` for full technical details.
